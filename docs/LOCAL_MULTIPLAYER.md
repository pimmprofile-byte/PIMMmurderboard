# 로컬(와이파이) 다중 플레이 방식

각 참가자가 **자기 PC/폰**으로 접속해 함께 논다. 별도 클라우드 배포 없이, **호스트 1대가
로컬 서버를 켜고** 같은 와이파이의 기기들이 붙는다. 이미 구현·검증된 구조다.

---

## 1. 구조

```
[내 폰] ─┐
[내 PC] ─┼─(같은 와이파이, HTTP 폴링)─▶ [호스트 PC: server.py]  ─▶  Claude API / Ollama
[친구폰]─┘                                 · 방(room) 상태 보유         (AI 배역 연기·투표)
                                           · 배역 클레임/동기화
                                           · 비밀은 소유 기기에만 전송
```

- **서버 권위(single source of truth):** 테이블 로그·배역 배정·투표·라운드는 전부 서버 메모리의
  한 '방'에 있다. 각 기기는 **1.5초 폴링**으로 최신 상태를 받아 화면을 갱신한다.
- **비밀 프라이버시:** 각 배역의 대본(죄·목표)은 `/api/sheet/{role}?clientId=` 로, **그 배역을 맡은
  기기만** 받을 수 있다(소유 검증, 남은 403). 히든롤 게임에 필수.
- **AI 좌석:** 사람이 안 맡은 배역은 'AI로 채우기'. 누구나 'AI 배역 부르기'로 발언을 트리거(페이스 조절).

## 2. 실행 & 접속 (호스트)

```bash
# 호스트 PC 1대에서
cd PIMMmurderboard
cp .env.example .env      # ANTHROPIC_API_KEY 입력 (무료로는 LLM_BACKEND=ollama)
pip install -r requirements.txt
python server.py          # 또는 ./run.sh (mac/linux) · run.bat (windows)
```

켜지면 접속 주소 2개가 출력된다:
```
이 컴퓨터      →  http://127.0.0.1:8790
같은 와이파이   →  http://192.168.x.x:8790   ← 폰·다른 PC는 이 주소로
```
- 서버는 `0.0.0.0`에 바인딩되어 LAN의 다른 기기가 접속 가능(`.env`의 `REUNION_HOST`).
- 각자 접속 → 배역 선택 → 빈 자리 AI로 채우기 → 플레이.

## 3. 접속이 안 될 때 (체크리스트)

1. **같은 와이파이**인가(게스트망 분리 주의). 
2. **방화벽**이 8790 인바운드를 막는가 → 허용. (mac: 시스템설정>네트워크>방화벽, win: 인바운드 규칙)
3. 주소가 `192.168…`/`10.…` 사설 IP가 맞는가(서버 출력값 사용).
4. 폰 브라우저에 `http://IP:8790` 정확히 입력(https 아님).

## 4. LAN 밖(원격)에서 붙이려면

같은 와이파이가 아니면, 호스트를 **임시 공개 URL로 터널**한다(코드 수정 없음):
```bash
# 예: cloudflared (또는 ngrok http 8790)
cloudflared tunnel --url http://localhost:8790
```
출력된 공개 URL을 참가자에게 공유. **API 키는 여전히 호스트 서버에만** 있으므로 안전.
> 상시 서비스가 필요해지면 그때 호스팅 배포를 검토(키는 서버측 유지 원칙 동일).

## 5. 현재 한계 / 다음 단계

- **단일 방(room):** 데모용으로 서버당 방 1개. 여러 방이 필요하면 room id 도입.
- **폴링(1.5s):** 데모엔 충분. 더 즉각적 반응이 필요하면 WebSocket/SSE로 승급 가능.
- **상태 휘발:** 서버 재시작 시 방 초기화(메모리). 필요 시 파일/DB 영속화.
- **동시성:** AI 발화는 1건씩 순차(typing 락). 여러 AI 동시 발화가 필요하면 큐잉 개선.

이 모두 현재 인프라 위에서 점진 확장 가능하며, 오늘 데모엔 위 구성이면 충분하다.
